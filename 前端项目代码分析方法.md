# 前端项目拆解与代码分析指南

## 一、项目整体认知

拿到一个前端项目，首先从宏观到微观建立认知：

**第一步：看目录结构**
```
project/
├── src/              # 源码核心
├── public/           # 静态资源
├── package.json      # 依赖和脚本入口
├── tsconfig.json     # TS 配置
├── vite.config.ts    # 构建工具配置
└── .env.*            # 环境变量
```

**第二步：读 `package.json`**，这是项目的"身份证"：
- `dependencies` → 运行时依赖，判断技术栈（React/Vue/框架版本）
- `devDependencies` → 工具链（构建、测试、Lint）
- `scripts` → 了解 `dev / build / test / lint` 命令

| 文件/依赖 | 判断依据 |
|---|---|
| `react` / `vue` / `svelte` | UI 框架 |

## 三、代码分析的核心维度

### 1. 入口链路追踪
从启动点开始向下追踪：
```
index.html
  └── main.tsx          # 应用挂载
        └── App.tsx     # 根组件
              └── Router → Pages → Components
```

### 2. 路由结构分析
路由文件揭示整个应用的页面地图：
```ts
// 关注：嵌套路由、懒加载、权限守卫
const routes = [
  { path: '/dashboard', component: lazy(() => import('./pages/Dashboard')) },
  { path: '/user', children: [...] }
]
```

### 3. 组件拆解方法
按职责分类分析组件：

```
components/
├── ui/          # 原子组件（Button, Input, Modal）
├── layout/      # 布局组件（Header, Sidebar）
├── business/    # 业务组件（UserCard, OrderList）
└── pages/       # 页面级组件
```

分析每个组件时关注：
- **Props 接口** → 组件的输入契约
- **内部状态** → 自管理的数据
- **副作用** → useEffect / watch 中的异步逻辑
- **事件输出** → emit / callback 的通信方式

### 4. 状态管理分析
```
Store 分析要点：
├── state 结构     → 全局数据模型
├── actions/mutations → 数据变更逻辑
├── 异步流程       → thunk / saga / async action
└── 与组件的连接   → useSelector / mapState
```

### 5. 数据流向梳理
```
用户操作 → 事件处理 → 状态更新 → UI 重渲染
              ↓
          API 调用 → 数据转换 → 存入 Store
```

## 四、代码质量分析

### 静态分析工具
```bash
# 依赖分析
npx depcheck              # 找出未使用的依赖
npx madge src/index.tsx   # 生成依赖图

# 复杂度分析
npx eslint src --ext .ts,.tsx

# 包体积分析
npx vite-bundle-visualizer   # Vite 项目
npx webpack-bundle-analyzer  # Webpack 项目

# 重复代码检测
npx jscpd src/
```

### 关键质量指标
| 指标 | 健康值 | 工具 |
|---|---|---|
| 组件行数 | < 300 行 | 手动统计 |
| 函数圈复杂度 | < 10 | ESLint |
| 测试覆盖率 | > 60% | Jest/Vitest |
| 重复代码率 | < 5% | jscpd |
| 包体积 | 按需评估 | bundle analyzer |

## 五、性能分析

### 运行时性能
```bash
# Chrome DevTools
Performance 面板 → 录制交互 → 找 Long Tasks
Lighthouse → 跑评分报告（FCP / LCP / CLS / TTI）
```

### 代码层面检查点
- **不必要的重渲染** → React.memo / useMemo / useCallback 缺失
- **大列表未虚拟化** → 缺少 react-window / virtual-scroller
- **图片未优化** → 缺少懒加载、WebP 格式、尺寸适配
- **代码分割** → 路由是否做了 lazy load
- **重复请求** → 相同接口是否有缓存策略

## 六、系统化分析流程（实操步骤）

```
① 克隆项目 → 跑起来先
   npm install && npm run dev

② 阅读文档 → README / CHANGELOG / wiki

③ 依赖扫描 → package.json + depcheck

④ 入口追踪 → index → App → Router

⑤ 页面地图 → 梳理所有路由

⑥ 组件树   → 每个页面的组件层级

⑦ 数据层   → Store 结构 + API 封装

⑧ 工具链   → 构建配置 + 环境变量

⑨ 质量扫描 → ESLint + 覆盖率 + 包体积

⑩ 输出文档 → 架构图 + 问题清单 + 改进建议
```

## 七、输出产物建议
